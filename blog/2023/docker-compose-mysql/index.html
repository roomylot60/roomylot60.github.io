Docker 명령어

docker run --platform linux/amd64
-p 3306:3306
--name [컨테이너 이름]
-e MYSQL_ROOT_PASSWORD=[루트 유저 비밀번호]
-e MYSQL_DATABASE=[데이터베이스 이름]
-e MYSQL_USER=[유저 이름]
-e MYSQL_PASSWORD=[비밀번호]
-d mysql

-f 옵션

일반적으로 docker-compose.yml 파일이 현 디렉토리 내에 있을 경우에 docker-compose up 명령어를 통해 실행할 수 있다. 다른 파일명이나 다른 경로의 파일을 통해 실행하고자 할 경우에 -f옵션을 사용한다.

$ docker-compose -f docker-compose-local.yml up

docker-compose-local.yml 파일을 docker-compose up

$ docker-compose -f docker-compose.yml -f docker-compose-test.yml up

-f 옵션을 통해 복수의 설정파일의 실행또한 가능하다. 이때, docker-compose.yml 파일 실행 후 docker-compose-test.yml 을 실행한다.

-d 옵션

docker-compose up 을 통해 설정파일을 실행할 때, Crtl + c 를 통해 모든 서비스를 종료할 수 있는데, 이를 막기 위해 설정파일을 백그라운드로 실행하기 위한 옵션이다.

$ docker-compose up -d

docker-compose.yml 파일 내에서의 명령어

version: '3'
services:
  db:
    image: mysql:[mysql버전]
    container_name: [컨테이너 이름]
    restart: always
    port:
      - [로컬 포트]:3306
    environment:
      MYSQL_ROOT_PASSWORD: [루트 유저 비밀번호]
      TZ: Asia/Seoul
    volumes:
      - ./db/mysql/data:/var/lib/mysql
      - ./db/mysql/init:/docker-entrypoint-initdb.d
    platform: linux/x86_64

  1. 컨테이너 접속 및 shell 생성

docker exec -it django bin/sh
  • sh는 Bourne shell로 apt 실행은 가능하나 내부에서 source 명령어 실행이 안되어 중간에 변경

  1. Django 세팅

cd [생성된 위치]

SECRET_KEY = ‘시크릿키’
